636. 132 模式

给你一个 n 个整数的序列 a1,a2,…,an，一个 132 模式是对于一个子串 ai,aj,ak，满足 i < j < k 和 ai < ak < aj。设计一个算法来检查输入的这 n 个整数的序列中是否存在132模式。
n 会小于 20,000。

参考:132模式

考虑一下方法，首先我们寻找i < j < k满足ai < ak < aj的式子，考虑这种情况，假如说有i < j < k满足ai < ak < aj，假如说i的下一个i+1有a(i+1) < ai，那么无疑i+1也满足上式，由此可见i应该是一个连续序列中的最小值，换句话说，我们寻找的i是i=0为初始态，若ai < a(i+1)则i++，否则就是我们所需要的i，然后在我们选择的i后面选择j，aj为最大值，那么同理，找到连续序列中的最大值，然后在确定的j后面寻找k，若没找到，则设新的i起始点为j+1

改进方法:采用栈做优化，方法如下:
首先我们考虑我们需要的i<j<k有ai<ak<aj，那么不妨这样考虑，我们逆序考虑，首先找一个变量做k，初始值为最小值，然后从后向前查找，对于每一个值，假如这个值小于k，那么可以认为已经找到了，原因是我们的初始值是最小值，假如新的值小于k，说明k已经被赋值了，而且k假如被赋值，栈中一定有值，我们知道栈中的值都是大于k的，那么三个值都凑齐了。如果新值不小于k，那么将其与栈顶对比，若大于栈顶，则将栈顶赋给k，然后将栈顶pop，然后push新值。我们考虑栈中数据特性，首先，栈中数据一定都大于当前的k，此外栈中数据从大到小排列，并且时间从后向前，也就是栈顶的数据一定是最小的也是最新的，那么我们可以想在这个更新过程中，k一定是逐渐变大，栈中也是逐渐更优的序列，而k越大，那么成功找到i的概率也就越高

index|0 | 1 | 2 |3 | 4 | 5 |6
|———|———|———-|———|———|———-|———|———
data|1 | 2 | 3 | 4 | 5 | 6 |0

以上述列表为例子，在每次压入数据都没问题，在到达我们最后自己压入的那个0式会进行处理，首先会计算一个area=(6 * (6-4-1)) = 6，原因就在于我们是先出栈再进行计算，所以计算结果上宽会多一个，这也就是为什么要减一，那为什么要先出栈再计算呢，原因在于我们需要计算当栈最后为空也就是栈深就是宽的情况，假如是先计算再出栈，那么就无法计算全部数据都出栈的情况了。

考虑本题，要求O(logN)的话就是考虑二分，由于A[0]<A[1]，A[n-2]>A[n-1]，也就是说中间肯定有一个峰值，我们采用二分法，假如说定到的中间点是峰值就返回，假如不是峰值，如果三个值是递增的，那么说明峰值在后半段，如果递减，则在前半段，如果为低谷值，则前后都有峰值任意选一个方向就可以。

735. 替换为右侧最大值

给一整数数组, 用当前元素之后数组中的最大元素来替换当前元素(右侧的最大元素). 因为最后一个元素的右边没有元素了, 所以用 -1 来替换这个值. 举个例子, 如果数组为 [16,17,4,3,5,2], 那么它就需要修改为 [17,5,5,5,2,-1].
DP，从右向左依次找最大值，对于每一个位置，他右侧的最大值是右侧更新后的值和本身值的最大值，进行运算，注意一点，最后一个-1不参与运算，换言之，假如最后一个值本身为-10，更新后为-1，那么倒数第二个值不应取两者的最大值-1，而应该取-10

可发现上面这种写法太过于丑陋，我们可以采用第一次判断在循环外进行优化，直接将data设为应该赋给的最大值而不是上一个的初始值，这样在循环外的第一次判断可以直接找到正确值从而省略掉判断
